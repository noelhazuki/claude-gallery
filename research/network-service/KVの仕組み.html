RES-011・Cloudflare KV設計メモ

KVってこういうもの

コインロッカーで考えるとわかりやすいで
倉庫には、こういうロッカーがいっぱい並んでる

KV（Key-Value）は、ラベル付きのロッカーがずらっと並んでる倉庫みたいなもんやと思ってや。1個のロッカーには「ラベル（key）」と「中身（value）」がセットで入ってる。

ロッカーの数は増やせる。でも1個のロッカーの中で「さらに細かく仕分け検索する」みたいなことはできへん。

1個開けてみると
🏷 dev_dashboard_v2
↓ このラベルで呼ぶと、中身が丸ごと出てくる
{ "items": [ ... 全アプリのデータ ... ], "logs": [ ... ] }
key
ラベル（名前）
value
中身（データ）
データベースとは別物

「データを保存する」って聞くとデータベースっぽく感じるけど、KVでできることはかなりシンプルや。

○
「このラベルの中身くれ」が一発でできるget（読む）・put（書く）だけ。速くて単純
×
「中身を見て条件で探す」はできないSQLの WHERE みたいな検索機能は無い
×
「ロッカー同士を紐付ける」もできないJOINみたいな結合機能も無い
📦 今回ダッシュボードで決めた形

同時に書き込みに来る人や仕組みが無いから、ロッカーは1個だけにして、今のlocalStorageと同じ「全部入りJSON」をそのまま入れる方針やったな。

dev_dashboard_v2
アプリ一覧 ステータス ログ 優先度 … これ全部、ひとまとめで1個のロッカーに入る
