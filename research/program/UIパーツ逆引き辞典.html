UIパーツ逆引き辞書 UI COMPONENT DICTIONARY

見つからへんかった…別の言葉で試してみてな

入力・操作系

Buttonボタン

別名: CTAボタン / ゴーストボタン（枠線だけのやつ）

押すと何かが起きる基本パーツ。塗りつぶし＝主役の操作、枠線だけ＝控えめな操作、という使い分けが定番。

FABフローティングアクションボタン

別名: フロートボタン / 右下の丸いやつ

画面の右下に浮いてる丸ボタン。「新規追加」みたいな、その画面で一番よく使う操作を1個だけ置く。

Toggle Switchトグルスイッチ

別名: スイッチ / オンオフ切替（タップで動くで）

ON/OFFを即時に切り替えるやつ。チェックボックスと違って「押した瞬間に反映される」設定向き。

Checkbox / Radioチェックボックス / ラジオボタン

別名: ラジオ＝丸ぽち

四角＝複数選べる、丸＝どれか1つだけ。この使い分けはUIの世界共通ルール。

Text Fieldテキストフィールド

別名: 入力欄 / インプット / テキストボックス

文字を打ち込む欄。上の小さい文字は「ラベル」、薄く入ってる例文は「プレースホルダー」って呼ぶ。

Select / Dropdownセレクトボックス

別名: プルダウン / ドロップダウンメニュー

タップすると選択肢のリストが垂れ下がってくる。選択肢が多い時にラジオボタンの代わりに使う。

Sliderスライダー

別名: レンジ / つまみ

つまみをドラッグして数値を調整。音量や明るさみたいに「だいたいこのへん」で決める値に向いてる。

🔍 妖怪を検索…

Search Bar検索バー

別名: 検索窓 / 検索ボックス

丸っこい形＋虫眼鏡アイコンが定番の型。このページの上にあるやつもこれ。

和食洋食中華

Chipチップ

別名: タグ / フィルターチップ

小さい丸角のラベル。タップで選択して絞り込みに使うことが多い。表示専用のものは「タグ」と呼ばれがち。

Segmented Controlセグメンテッドコントロール

別名: セグメント切替 / 表示切替ボタン

くっついたボタン群からどれか1つを選ぶ。タブより軽い「表示モードの切り替え」に使う。iOS出身のパーツ。

Date Pickerデートピッカー

カレンダーから日付を選ぶ入力パーツ。手打ちより入力ミスが減る。時刻版は「タイムピッカー」。

Ratingレーティング

別名: 星評価 / スター評価

星の数で評価を入力・表示する。5段階が定番。表示専用と入力可能の両方の使い方がある。

別名: 引っ張って更新

一覧を下に引っ張ると最新に更新される操作。Twitter発祥で今やスマホの常識。PWAでも実装できる。

Swipe Actionスワイプアクション

別名: スワイプ削除 / スワイプメニュー

リストの行を横にスワイプすると削除や完了ボタンが現れる。メールアプリでおなじみの操作。

Tabsタブ

別名: タブバー / タブ切り替え（タップで動くで）

同じ画面の中でコンテンツを切り替える。選択中は下線や色で示すのが定番。

Hamburger Menuハンバーガーメニュー

別名: 三本線 / メニューアイコン

三本線がハンバーガーに見えるからこの名前。押すと横からドロワーが出てくる組み合わせが多い。

Drawerドロワー

別名: サイドメニュー / スライドメニュー

画面の横からスライドして出てくるメニュー。引き出し（drawer）が語源。奥の画面は半透明の黒で暗くする。

Bottom Navigationボトムナビゲーション

別名: ボトムナビ / 下タブ / タブバー(iOS)

スマホアプリの画面下に固定される主要メニュー。3〜5個が適正。親指で届く位置なのがミソ。

Breadcrumbパンくずリスト

別名: パンくず

今どの階層にいるかを示す道しるべ。ヘンゼルとグレーテルのパンくずが由来。ポータルサイトと相性◎。

Paginationページネーション

別名: ページ送り / ページャー

一覧を複数ページに分けて移動する。スマホでは代わりに「無限スクロール」を使うことも多い。

Kebab Menuケバブメニュー

別名: 縦三点 / オーバーフローメニュー（横並び「…」はミートボールメニュー）

「⋮」を押すと補助メニューが出る。串刺しのケバブに見えるのが由来。よく使わへん操作の隠し場所。

Step Indicatorステップインジケーター

別名: プログレストラッカー / ウィザードステップ

「入力→確認→完了」みたいな複数画面の手続きで、今どこかを示す。数量の＋−（ステッパー）とは別物なので注意。

Cardカード

別名: タイル / パネル

画像＋テキストを1枚にまとめた枠。角丸＋薄い影が定番。妖怪プランナーのタスク表示もこれやね。

タップで開いたり閉じたりする。楽器のアコーディオンが由来。

Accordionアコーディオン

別名: 折りたたみ / 開閉パネル（タップで動くで）

見出しをタップすると中身が展開する。FAQや詳細設定みたいに「普段は隠しときたい」情報に使う。

別名: 空状態 / ゼロ件表示

Dividerディバイダー

別名: 区切り線 / セパレーター

コンテンツの切れ目を示す線。文字入りのものはログイン画面の「または」でよく見る。

Table / Data Gridテーブル

別名: 表 / データグリッド（並べ替えや編集ができる高機能版がデータグリッド）

行と列でデータを見せる基本形。スマホでは横に入りきらんので、カード表示に変換する設計も多い。

Lightboxライトボックス

別名: 画像ビューア / フォトビューア

サムネイルをタップすると、暗い背景の上に画像がドーンと拡大表示される。ピンチ拡大とセットが定番。

Modalモーダル

別名: ダイアログ / ポップアップ / 確認画面

画面の中央に被さって、答えるまで他の操作をブロックする窓。背後の暗い部分は「オーバーレイ」。

Toastトースト

別名: トースト通知（トースターから飛び出すイメージ）

画面の端にポンと出て、数秒で勝手に消える通知。操作を邪魔しないのがモーダルとの違い。

1件削除しました

Snackbarスナックバー

別名: アクション付きトースト（Android/Material用語）

トーストに「元に戻す」みたいなボタンが付いたやつ。削除の取り消し導線として超定番。

設定を開きます

Tooltipツールチップ

別名: 吹き出しヒント / ホバーヒント

アイコンに触れると出る小さな補足。PCのホバー前提なので、スマホでは長押しか別の手段に置き換える。

Progress Barプログレスバー

別名: 進捗バー / ローディングバー

進み具合を横棒で見せる。残り時間の見当がつく処理に使う。円形のものは「プログレスリング」。

Spinnerスピナー

別名: くるくる / ローディングインジケーター

「待っててな」のくるくる。終わりが読めない処理に使う。読めるならプログレスバーの方が親切。

Skeleton Screenスケルトンスクリーン

別名: スケルトン / 骨組み表示

読み込み中にコンテンツの形だけ灰色で先に見せる手法。スピナーより体感待ち時間が短くなる。

⚠️ ネットワークに接続できません

Banner / Alertバナー / アラート

別名: お知らせ帯 / 警告表示

画面の上部などに帯状で出る通知。トーストと違って消えずに残り、対応が必要な状態を知らせる。

📋 詳細を見る
✏️ 編集する

画面の下からせり上がるパネル。上に引っ張ると全画面まで広がるタイプもある。地図アプリでおなじみ。

下から出てくる操作の選択肢リスト。危険な操作は赤字、キャンセルは分離して置くのがiOSの流儀。
