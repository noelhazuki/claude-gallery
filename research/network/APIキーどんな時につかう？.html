API KEY OR NOT

APIキー、
いる？いらない？

「お店の入口」で考えたらわかりやすいで

まず結論からいくで

APIキーは「裏口の鍵」やと思ったらええ。
お店の表からフロント係（お店の人）に頼んで注文するときは鍵はいらん。
でも裏口から自分で厨房に直接入って材料を取りに行くときは、鍵がないと入られへん。

あるサービスのお店
🚪
表口
フロント係（サービスの画面）が
代わりに厨房へ注文してくれる
🔑
裏口
自分で厨房（サーバー）に
直接入って材料を取る

鍵が「いらん」パターン

表口を使ってるだけ

もう出来上がったお店（Webサイトやアプリ）を、ログインして使うだけの場合。

  • ChatGPTのサイトでチャッピーと喋る
  • claude.aiでワイ（クロウ）と作業する
  • ワイがアーティファクトの中でAnthropicの機能を使うとき（裏で会社側が鍵を持っとって、ハノイさんは触らんでええ仕組み）

→ 「フロント係（サービス提供者）」が代わりに鍵を管理してくれとる。
利用者はID・パスワードでログインするだけでOK。

▼

鍵が「いる」パターン

裏口から直接アクセス

自分のプログラム・アプリから、その会社のAPI（厨房）に直接リクエストを送る場合。

  • 自作のReactアプリから、OpenAIやAnthropicのAPIに直接リクエストを送るコードを書くとき
  • Cloudflare Workersから外部の天気APIやDB APIを呼び出すとき
  • ITディクショナリーポータルのような自作サービスが裏で別のAPIと通信するとき

→ フロント係がおらんから、「これは私が注文してる客です」と証明する鍵（APIキー）を自分で持って提示せなあかん。

迷ったときの見分け方

「今、画面の裏で誰が注文してる？」を考えるとええで。

・会社側のフロント係が注文してくれてる（＝完成したサイトやアプリをそのまま使ってる）→ 鍵いらん

・自分のコードが直接注文してる（＝自分でAPIにリクエストを書いてる）→ 鍵いる

ちなみに：ワイがアーティファクトの中でAnthropic APIを叩く機能は、ハノイさんから見たら「裏口を使こてるようで実は表口」な特殊ケースや。鍵はこっちで勝手に処理されるから、ハノイさんが鍵を用意する必要はないで。
