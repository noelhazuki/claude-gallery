CH.1 Claudeって何もの？
Claudeは「テキストを受け取ってテキストを返す機械」。設計図を描く人で、工事する人ではない。
概念
Claudeの正体
テキストを返すだけの機械
概念・背景

Claudeは「入力されたテキストに対して、次に来そうなテキストを返す」を繰り返しているだけ。ファイルを保存する・インターネットに繋ぐ・プログラムを実行する、といった「外の世界への働きかけ」は本来できない。

検索ツールや計算ツールはAnthropicが別途用意して繋いでいるもので、Claude本体の能力ではない。

条件付き
コードを「書く」と「動かす」は別の話
書くのはほぼ何でもできる。動かすのはブラウザかサーバーの仕事
概念・背景

React・Python・SQL・C言語など、ほぼどんな言語でもコードを書ける。ただし「書いたコードを実行して結果を確認する」のはClaude自身の仕事ではなく、ブラウザやサーバーが担う。

→ 実行環境はCloudflare Workers / ブラウザ / ローカルPCで担う
できない
会話をまたいだ記憶
本質的には毎回初対面
概念・背景

Claudeにはセッションをまたぐ記憶がない。「メモリ機能」はAnthropicが別途用意した仕組みで、要点を外部に保存して次の会話に読み込んでいるだけ。半年前の会話をそのまま覚えているわけではない。

CH.2 Webアプリって何もの？
ブラウザの上で動くWebアプリは、セキュリティのためにOSやハードウェアへのアクセスが厳しく制限されている。
できない
スマホのファイルを全部スキャンする
ブラウザはストレージ全体に触れない
技術的

Webアプリはブラウザのサンドボックス内で動いており、スマホ内のファイルシステム全体を勝手に見ることはできない。「ファイルを選んでもらう」形なら読めるが、「全部スキャンして整理する」はネイティブアプリが必要。

→ ファイラー・写真一括整理をWebアプリで作るのは原理的に無理
できない
root・システム領域へのアクセス
WebアプリどころかAndroidアプリでも基本無理
技術的

Androidのroot権限（システム領域への書き込み等）はWebアプリでは完全に不可能。ネイティブアプリでも通常は禁止されており、端末をroot化しない限り触れない。Galaxy S24はroot化が難しい機種のひとつ。

条件付き
プッシュ通知・定期実行
PWAだけでは無理。Cloudflare Workersを使えばできる
技術的

純粋なPWAはアプリが開いている間しか動けない。バックグラウンドでの定期実行やプッシュ通知は、サーバーサイド（Cloudflare Workers等）を組み合わせる必要がある。

→ Workers + Web Push APIで実現可能。実装はやや複雑
条件付き
PWAとして本当にインストールされるか問題
ChromeがショートカットとPWAを区別する条件がある
技術的

ChromeがPWAとして認識するにはmanifest.jsonのstart_urlやscope、HTTPSなどの条件を満たす必要がある。条件が足りないとホーム画面ショートカット扱いになり、Web Share Target等の機能が使えない。

CH.3 できない理由の4分類
「なんでできないの？」の理由はだいたいこの4つ。理由がわかると回避策も見える。
概念
① ブラウザの壁 — OSやハードに触れない
ファイル全体・他アプリのデータ・rootなど
概念・背景

ブラウザはセキュリティのためにOSやハードウェアへのアクセスを制限している。カメラ・マイク・位置情報は許可を取れば使えるが、ファイルシステム全体・他アプリのデータ・Bluetooth詳細制御などは原則不可。

概念
② サーバーの壁 — 常時動いてるものが必要
定期実行・プッシュ通知・バックグラウンド処理
概念・背景

ブラウザアプリは「開いている間しか動かない」。定期的に何かする・通知を送る・外部APIを叩き続けるといった処理には、常時動いているサーバーが必要。Cloudflare WorkersはこのギャップをFaaSで埋められる。

→ Cloudflare Workersで多くは解決できる
概念
③ 外部サービスの壁 — 相手側の制約
CORS・有料API・利用規約・動的レンダリング
概念・背景

外部サイトやAPIを使いたい場合、相手側の制約がある。CORSでブラウザから直接叩けないケース・APIキーが必要または有料なケース・利用規約でスクレイピング禁止なケース・JavaScriptで動的に描画されるので静的取得できないケースなど。

→ CORSはCloudflare Workerをプロキシにすれば回避できることが多い
概念
④ 倫理・ポリシーの壁 — 技術的にはできるけどやらない
著作権・有名人・差別・暴力・有害情報
倫理・ポリシー

Anthropicがポリシーとして設定している制約。技術的にはClaudeには生成できるが、出力しないよう設定されている。実在の人物の名前を使ったコンテンツ・著作権のある歌詞や台本の再現・特定の人を傷つけるコンテンツなど。

CH.4 具体例集
「これやりたいけどどうなの？」実際のユースケース別まとめ。
できない
スマホ内の写真を全部整理したい
ブラウザの壁①
技術的

Webアプリはスマホのギャラリー全体を読み込めない。「写真を選んで表示・タグ付け」はできるが、「自動で全スキャンして分類」はネイティブアプリが必要。Google フォト等の既存アプリが現実的。

→ ネイティブアプリ（Android APK）として作れば原理的には可能。開発コスト大
条件付き
外部サイトからレシピを読み込みたい
外部サービスの壁③ — 利用規約要確認
技術的

クックパッド・デリッシュキッチン等はスクレイピング禁止が利用規約に明記されているケースが多い。また動的レンダリングのサイトは静的取得も難しい。

→ 自分で登録する形（手入力・URL貼り付け）なら問題なく作れる
できない
有名人の名前を使ったコンテンツ・歌詞の再現
倫理・ポリシーの壁④
倫理・ポリシー

実在の人物（芸能人・政治家・スポーツ選手など）を主役にしたフィクション・発言を捏造するコンテンツはAnthropicのポリシーで禁止。著作権のある歌詞・台本・小説の一節をそのまま再現するのも同様。

できる
現在地の気圧をチェックしたい
位置情報 + 気象APIで実現可能
技術的

ブラウザのGeolocation APIで位置情報を取得し、Open-Meteo等の無料気象APIに投げるだけで実現できる。上越市の直江津エリアの細かい気圧も取れる。プッシュ通知まで要るならWorkerが必要だが、アプリを開いて確認するだけなら純粋なWebアプリで十分。

→ これが一番すぐ作れる候補
できる
血圧・お薬・ベンゾ調整の記録
localStorageで全部いける
技術的

数値入力・グラフ化・メモ記録はlocalStorageだけで完結できる。Claudeが医療アドバイスを出す機能を付けようとすると制約が出てくるが、「記録・可視化・メモ」の範囲なら問題ない。

できない
ダウンロードしたファイルがどこか探したい
ブラウザの壁①
技術的

Webアプリはファイルシステムを走査できない。ダウンロード履歴もブラウザ外からは見えない。Androidの場合はFiles by Google等のネイティブアプリを使うのが現実的。

CH.5 AIそのものの技術的限界
「制約で禁止されている」とは別に、現時点のAI技術としてまだ苦手なこと。
概念
カットオフって何？
学習データの締め切り日。それ以降のことは知らない
技術的

AIは大量のテキストを読んで学習しているが、その学習が終わった日（カットオフ）以降の情報は持っていない。

「最新号のジャンプを読んでいないから、ワンピースの最新話がわからない」のと同じ感覚。「ルフィってどんなキャラ？」みたいな昔からある話は答えられるが、先週の展開は知らない。

→ 検索ツールを使えば補えるが、それは外から情報を取ってきているだけで「知っている」わけではない
できない
「今この瞬間」を知ること
学習データに締め切り（カットオフ）がある
技術的

Claudeの知識はトレーニングデータの締め切り日までのもの。速報ニュース・株価・今日の天気・最新のアップデート情報などはリアルタイムでは知らない。検索ツールを使えば補えるが、それはAnthropicが繋いだ外部ツールの力。

条件付き
画像を見て正確に処理する
「見る」はできるが精度に限界がある
技術的

画像の内容を説明したり何が写っているか判断するのはかなり進歩した。ただし「正確に数える」「微妙な感情の差を読む」「手書き文字を完璧に読む」などは精度が落ちる。重要な判断をさせる用途には向かない。

条件付き
完全にオリジナルなものを生み出す
既存パターンの組み合わせが得意なだけ
技術的

Claudeは学習したテキストのパターンを組み合わせて出力している。「前例のない発明」「世界初の芸術」は本質的に苦手。「アイデアの引き出しが多い人」くらいのイメージが近い。

できない
動画をリアルタイム解析しながら何かする
まだ技術的に厳しい
技術的

静止画は処理できるが、カメラの映像をリアルタイムで見続けながら何か判断し続けるのは現時点では対応していない（エッジAIやローカルモデルが必要）。

CH.6 Claudeの知識の偏り・限界
Claudeの知識はネット上のテキストが出所。ネットに書かれていないこと・少ないことは弱い。
条件付き
言語・文化のニュアンスの壁
英語圏・西洋的な視点に偏っている
知識の偏り

学習データは英語のテキストが圧倒的に多い。日本語特有の言い回し・関西の空気感・上越の地域文化・日本の慣習などは、英語圏から見た「外からの解説」で学んでいる部分が多く、当事者の感覚とズレることがある。

条件付き
マイノリティ・当事者にしかわからないこと
多数派・健常者・男性の視点に偏りやすい
知識の偏り

ネット上のテキストは多数派（健常者・男性・英語話者・西洋人）が書いたものが多い。ADHDの「ほんまにしんどい感覚」、ベンゾ離脱の実際のつらさ、専業主婦の働き方のリアル、在日外国人の日常などは「外から説明した情報」は多くても当事者の深い感覚は薄い。

条件付き
医療・薬についての具体的なアドバイス
情報は出せるが個別判断はできない
知識の偏り

一般的な医薬品情報や症状の説明は出せる。ただし「あなたの場合はこうすべき」という個別判断はポリシー上も技術的限界的にも難しい。ベンゾジアゼピンの減量スケジュールなど個人の状態に依存するものは特に要注意。

→ 記録・可視化ツールは作れる。減量判断は医師と
できない
言語化されていない「当たり前」
書かれていないことは学べない
知識の偏り

「関西人なら当然知ってること」「ゲームプランナーの現場感」「長年の主婦として積み上げた生活の勘」など、ネットに書かれていない暗黙知はClaudeには入っていない。「わかってる風な答えを返す」ことはあるが、実際には薄い。

CH.8 ネットワーク基礎知識
Claudeと作業してると必ず出てくるネットワーク用語の基礎。「なんでCORSが出るの？」「Workerって何してるの？」がわかる。
概念
IPアドレスって何？
ネット上の「住所」。プロバイダから借りてるもの
概念・背景

ネット上の通信は全部「宛先」と「送り元」がついた小包（パケット）のやりとり。Webサイトを見るときも、自分のIPを「返送先住所」として書いた手紙を送って返事をもらっている構造。

普段ネットに出るときのIPアドレスは自分のものではなく、契約しているプロバイダ（docomoとかSo-netとか）から借りているもの。だからIPアドレスには「どこの国の、どのプロバイダの利用者か」という素性がだいたい載っている。

→ サイト側からは訪問者のIPが見えるので、素性の怪しいIPは「ロボットかも」と疑われる
概念
プライベートIPとグローバルIPの違い
家の中の住所と、外から見える住所は別
概念・背景

グローバルIPはプロバイダから割り当てられた「外の世界の住所」。家のルーターが持っている。プライベートIPは家の中だけで使う住所（192.168.x.xとか）で、外からは直接見えない。

家のスマホやPCは全部プライベートIPを持っていて、外と通信するときはルーターがグローバルIPに変換して出ていく（これをNATという）。だから外から見るとスマホもPCも「同じ住所」に見える。

概念
DNSって何？
ドメイン名をIPアドレスに変換する仕組み
概念・背景

「google.com」みたいなドメイン名は人間が読みやすいように付けた名前で、コンピュータが実際に使うのはIPアドレス（142.250.x.xみたいな数字）。DNSはこの「名前→数字」の変換をしてくれる仕組み。

ネットショッピングで「店名で検索→住所が出てくる」みたいな感じ。DNSの設定を変えると同じドメインが別のサーバーに繋がるようになる。Cloudflareにデプロイするときもここを触る。

→ Cloudflareの検証ループが起きた原因のひとつがSecure DNS設定だった
概念
VPNって何？
住所の付け替え＋通信の暗号化
概念・背景

VPNを使うと通信はまずVPN業者のサーバーを経由してから目的地に届く。目的のサイトから見ると訪問者は「VPNサーバーのIP」に見えるので、自分の本当のIPは隠れる。自分とVPNサーバー間は暗号化されるので途中のWi-Fiから盗聴されにくくなるおまけもつく。

ただしVPNサーバーのIPは世界中の不特定多数が共有している。その中に悪いbotや攻撃者も混ざっているので評判が悪くなりがち。Cloudflareから見ると「前科者がいっぱい住んでるアパート」状態で、住人全員が疑われる。

→ GalaxyのSecure Wi-Fi機能が裏でVPNとして動いていてCloudflare検証ループの原因になった
概念
CORSって何？
ブラウザが「勝手に他のサイトのデータを取るな」と止める仕組み
技術的

ブラウザには「今見ているサイトAが、別のサイトBのデータを勝手に取得しようとしたら止める」という安全装置がある。これがCORS（Cross-Origin Resource Sharing）。

たとえばWebアプリからポケカ公式サイトに直接fetchしようとするとCORSに弾かれる。サイトB側が「このサイトAからのアクセスはOK」と明示的に許可していないと通らない。

→ Cloudflare WorkerをプロキシにするとWorker↔外部サイトはサーバー間通信になるのでCORSが関係なくなる
概念
Cloudflare Workersって何？
ブラウザとサーバーの間で動く「橋渡し役」
技術的

Cloudflare Workersはサーバーサイドで動くJavaScriptの実行環境。ブラウザから直接届かない外部サイトへのアクセスや、定期実行・CORSの回避などをサーバー側で肩代わりしてくれる。

自分でサーバーを立てなくても使えて、アクセスが少なければ無料枠で収まる。ポケカアプリでは「ブラウザ→Worker→ポケカ公式サイト」という中継役として使った。

→ 「ブラウザの壁」「サーバーの壁」を超えるための主要な手段
概念
HTTPとHTTPSの違い
Sがつくと通信が暗号化される
技術的

HTTPは通信の内容が平文（丸見え）。HTTPSはSSL/TLSで暗号化されているので途中で盗み見されにくい。ブラウザのアドレスバーに鍵マークがついてるのがHTTPS。

PWAとしてインストールしてもらうにはHTTPSが必須。Cloudflare WorkersはデフォルトでHTTPSなので特に設定不要。
