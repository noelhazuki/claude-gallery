フロントエンド10年の変化（2014〜2024）

〜2013年
jQuery時代

HTML+CSS+jQueryで「ページを動かす」。サーバーがHTMLを生成してブラウザに返すのが主流。

2013〜2016年
React登場・普及期

FacebookがReactを2013年に公開。コンポーネント単位でUIを組み立てる概念が広まる。2016年頃から業界標準に。

2015〜2018年
Node.js・npmの爆発

JavaScriptがサーバーサイドでも動作可能に。npmで「部品を組み合わせる」文化が定着。Webpackによるバンドルが当たり前に。

2018〜2022年
JAMstack・API駆動

フロントとバックが完全分離（APIでつなぐ）。Cloudflare PagesなどCDN・エッジで動く時代に。ViteがWebpackを置き換え始める。

2023年〜現在
AI統合時代

GPT-4・Claude・GeminiのAPIが公開。フロントエンドから直接LLMを叩けるように。「AIをパーツとして組み込む」開発が標準化。

// HTMLを「誰が作るか」の変化 Perl時代 サーバーがHTMLを生成 → ブラウザは受け取るだけ React時代 サーバーはJSONだけ返す ブラウザ側のJSがHTMLを生成する // これが「フロントとバックの分離」の正体

インフラの進化（エッジ・CDN・サーバーレス）

速さの2つの要因

① 回線速度（光・5G）

データの「転送速度」。パイプの太さ。光回線で解決できる。

② 物理的な距離（レイテンシ）

データの「往復時間」。パイプの長さ。光速でも距離は縮まらない。東京↔NYで最低150ms。

エッジとは

従来 大阪のユーザー → 東京のサーバー（往復500km） エッジ 大阪のユーザー → 大阪のエッジ（ほぼゼロ距離） // Cloudflareは世界300都市以上にサーバーを設置 // 日本：東京・大阪・福岡・沖縄の4拠点（2025年時点）

サーバーレスとは

// サーバーがなくなったわけではない 従来 自分でサーバーを借りて管理する （電気代・メンテ・スペック選定…） サーバーレス Cloudflare等のサーバーを間借り コードだけ置けばあとは向こうがやってくれる

JAMstackとは

JavaScript + API + Markup の略。

従来のWordPress等 ブラウザ → サーバー → DB問い合わせ → HTML生成 → 返す ※毎回サーバーが動く・遅い JAMstack（Cloudflare Pages等） ビルド時にHTMLを全部作っておく ブラウザ → CDN（世界中に配置）→ 爆速 動的な部分だけAPIを叩く

APIとJSONの関係

// サーバーとAPIの関係 サーバー ＝ 建物（実行する場所） API ＝ 窓口（話しかけるルール） // サーバーの中にAPIがある フロント（React） ↓ APIを叩く（HTTPリクエスト） サーバー ├── APIの窓口 ← リクエストを受け取る ├── 処理する └── DBに問い合わせ・計算 ↓ JSONを返す { "name": "花子", "age": 5 }

REST APIのルール

メソッド意味例
GET取得GET /users/123
POST送信・作成POST /users
PUT更新PUT /users/123
DELETE削除DELETE /users/123

JSONとは

// CSSとの比較 CSS → 見た目の設定を外部に書く JSON → データの構造を外部に書く // JSONの例 { "name": "花子", "age": 5, "hobbies": ["お絵かき", "ねんど"] } // JavaScriptが読んでいる const user = JSON.parse(response) return <p>{user.name}</p> ← 画面に表示
JSONはXML（2000年代の重い形式）の代替として普及。JavaScriptのオブジェクト記法がそのままデータ形式になった。Perl時代には存在しなかった文化。
04

Flashの終焉とHTML5

2010年
ジョブズの宣言

「iPhoneにFlashは載せない」。理由：バッテリー消費・クラッシュ・セキュリティ穴。

2010〜2020年
HTML5による置き換え

動画・アニメーション・インタラクションがFlashなしで実現可能に。スマホ普及でFlash非対応端末が多数派に。

2020年12月31日
Adobe、公式サポート終了

2021年1月12日にFlash Player自体がFlashコンテンツの実行をブロック。文字通り「自分で自分を殺した」。

Flashがやっていたことの行き先

Flash時代現在
FlashアニメCSS animation・WebGL・Three.js
FlashゲームWebGL・Unity WebExport
動画プレイヤーHTML5のvideoタグ
インタラクティブ広告JavaScriptアニメーション
Flashは技術者からは「すごい」表現力を持っていたが、ユーザーには「邪魔・重い・クラッシュする」だった。技術の優秀さとユーザー体験は別物という教訓。

Pythonが台頭した理由

〜2016年 データ分析・統計 → Pythonが得意 Web開発 → JavaScript/Ruby/PHPの領域 2016〜2022年 機械学習ライブラリ（TensorFlow・PyTorch）が ぜんぶPython製 → AIやるならPython一択に 2023年〜現在 LLM（ChatGPT等）のAPIが公開 → Pythonでサクッと繋げる → データ加工・プロンプト管理・複数AI連携の 道具が全部Pythonに集中

LLM関連用語

LLM（Large Language Model）

ChatGPT・Claude等の「大規模言語モデル」の総称。

RAG（Retrieval-Augmented Generation）

自分のデータをAIに読ませて答えさせる仕組み。質問→関連ドキュメント検索→LLMに渡す→回答。

LangChain

API呼び出し→加工→別のAPIに渡す…を簡単に書けるフレームワーク。

LangGraph

AIが「考えながらループして判断する」エージェント向けフレームワーク。
